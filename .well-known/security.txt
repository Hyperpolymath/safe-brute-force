# Security Policy

SafeBruteForce is a security testing tool. We take security seriously.

# Reporting Security Vulnerabilities

If you discover a security vulnerability, please:

1. **DO NOT** create a public GitHub issue
2. Report privately via GitHub Security Advisories (preferred):
   https://github.com/Hyperpolymath/safe-brute-force/security/advisories/new
3. Or email: security@safeBruteForce.example (use PGP if possible)
4. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if known)

# Responsible Disclosure

We follow coordinated vulnerability disclosure:

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Status Updates**: Every 7-14 days
- **Fix Timeline**: 90 days maximum (preferably sooner)
- **Public Disclosure**: After fix is released and deployed
- **Credit**: We acknowledge security researchers (unless anonymity preferred)

# Scope

**In Scope**:
- Safety mechanism bypasses
- Authentication/authorization issues
- Code injection vulnerabilities
- Dependency vulnerabilities
- Information disclosure
- DoS vulnerabilities
- Unsafe LFE/Erlang code patterns

**Out of Scope**:
- Social engineering
- Physical security
- Third-party services/infrastructure
- Theoretical attacks without proof-of-concept
- Misuse by end users (report to abuse@safeBruteForce.example)

# Security Features

SafeBruteForce includes built-in security:

- Mandatory pause mechanism (cannot be fully disabled)
- Rate limiting
- Authorization verification
- Audit logging
- Input validation
- No unsafe code blocks
